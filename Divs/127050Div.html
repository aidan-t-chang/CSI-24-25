Introduction

Linked Lists are a fundamental data structure present in almost every programming language. Each linked list contains a value and a pointer towards the next node. The term "linked" comes from the fact that the values within the list are linked to each other, meaning it is a little harder to insert and remove elements, since all the values are connected to each other in a predetermined way. Linked lists were first used around the 1950s and ever since, they have been a key stepping stone to fundamentally understanding all data structures and algorithms.



Why Linked Lists?

Why choose a linked list instead of a regular list? Great question. While the strict methods of the linked list may seem detrimental, they are there for a reason. While programming the insertion and deletion for linked lists may seem unnecessarily hard, it allows for much faster time complexities for insertion and deletion. Linked lists are also good for dynamic programming situations, where the data structure's size must constantly change. Overall, linked lists are a powerful data structure that are used as the building block to many other data structures due to their efficiency compared to a regular unsorted, unorganized array.
